# Next.js 16 — What differs from your training data

This project uses **Next.js 16.2.3** with **React 19.2**. Your training data covers through Next.js 15. The breaking changes below are the ones most likely to affect this codebase.

## Async Request APIs (breaking — most common pitfall)

`cookies()`, `headers()`, `draftMode()`, route `params`, and page `searchParams` are now **fully async**. The synchronous compat shim from Next.js 15 is gone. Always `await` them:

```ts
// WRONG — was allowed in 15, throws in 16
const headersList = headers()

// CORRECT
const headersList = await headers()
```

Route handler `params` (dynamic segments) must also be awaited:

```ts
export async function GET(_req: NextRequest, ctx: RouteContext<'/api/[id]'>) {
  const { id } = await ctx.params   // must await
}
```

Use `PageProps<'/path/[slug]'>` and `LayoutProps` helpers (generated by `next typegen`) for type-safe async props in pages and layouts.

## Turbopack is the default bundler

`next dev` and `next build` now use Turbopack by default. No flag needed.

**Impact for this project:** `better-sqlite3` is a Node.js native module. It must only be imported in server-side code (route handlers, server components). If it ever ends up in a client bundle, Turbopack will error rather than silently stub it.

To opt out of Turbopack for a specific build: `next build --webpack`.

## `turbopack` config moved to top level

```ts
// Next.js 15
experimental: { turbopack: { ... } }

// Next.js 16
turbopack: { ... }   // top-level in next.config.ts
```

## `middleware` renamed to `proxy`

- File: `middleware.ts` → `proxy.ts`
- Named export: `export function middleware` → `export function proxy`
- Config key: `skipMiddlewareUrlNormalize` → `skipProxyUrlNormalize`

This project does not currently use middleware/proxy.

## Caching APIs

- `unstable_cacheLife` and `unstable_cacheTag` are now stable — drop the `unstable_` prefix.
- `revalidateTag` now requires a second `cacheLife` profile argument: `revalidateTag('tag', 'max')`.
- New `updateTag` (Server Actions only) for read-your-writes semantics.
- New `refresh()` from `next/cache` to refresh the client router from a Server Action.

## Partial Prerendering (PPR)

`experimental.ppr` is gone. PPR is now opt-in via `cacheComponents: true` in `next.config.ts`, but the model has changed — do not blindly migrate PPR config.

## React 19.2

Available: `ViewTransition`, `useEffectEvent`, `Activity`. React Compiler (`reactCompiler: true` in config) is stable but opt-in.

## What has NOT changed

- App Router file conventions (`page.tsx`, `layout.tsx`, `route.ts`, `loading.tsx`, `error.tsx`)
- Route Handler exports (`GET`, `POST`, etc.) and `Response.json()` usage
- Server vs. Client Component model (`"use client"`, `"use server"`)
- `next/navigation` hooks (`useRouter`, `usePathname`, `useSearchParams`)
- Tailwind CSS, TypeScript, and ESLint integration
